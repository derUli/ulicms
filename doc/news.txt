===== Neues in UliCMS 7 =====

[*] Neu: Alle Einträge aus Menü X nach Menü Y verschieben
[*] Enhancement: Konstante ULICMS_ROOT hinzugefügt
[*] Cache: Man kann zwischen Cache_Lite und File als Caching-Methode wählen.
[*] Cache auch per GUI einstellbar.
[*] Experteneinstellungen nun unter Sonstiges
[*] Speichern per Strg+S (optional)
[*] Option hinzugefügt "Standard-Gruppe für neue Nutzer"
[*] Implementation einer Datenbankabstraktion begonnen.
    Alle mysql_ Funktionen wurden durch db_ Wrapper-Funktionen die sich in der Datei "db_functions.php" befinden ersetzt.
	Eine für PostgreSQL und SQLite Datenbanken angepasste "db_functions.php" ist geplant.
	Es ist außerdem geplant, den Installer soweit anzupassen, dass man die zu nutzende Datenbanksoftware auswählen kann.
[*] Die Datenbank wird nun über mysqli_ angesprochen, da die alten mysql_ Schnittstelle in PHP 5.5
    als Deprecated (veraltet) gekennzeichnet ist. 
    Achtung! Diese Änderung hat zur Folge, dass Pakete, die noch die alte API nutzen, 
    nicht mehr funktionieren können.
    Bitte installieren Sie Pakete nur aus der offiziellen Paketquelle oder erstellen Sie mit ulicms-pkg eine Paketquelle, die mit dieser Version des CMS kompatibel ist.
[*] Die mysql_ Variablen in der Datei "cms-config.php" müssen bei einem Upgrade durch db_ ersetzt werden.
    z.B. mysql_password ersetzen durch db_password.
[*] Folge Konfigurationsvariable wurde hinzugefügt:
    $db_type
	Bei einem Upgrade des Systems muss folgende Zeile unter den MySQL-Zugangsdaten in der cms-config hinzugefügt werden:
	var $db_type="mysql";
[*] Neue API-Funktionen hinzugefügt:
    isModuleInstalled, clearCache, getOnlineUsers und weitere
[*] Diverse Funktionalitäten in Helper-Klassen ausgelagert, 
    Aus Kompatiblitätsgründen bleiben die Prozeduralen API-Funktionen erhalten.
[*] Access Control List hinzugefügt.
[*] Bei der Installation kann man den Namen des Adminaccounts ändern.
[*] Kategorien können hinzugefügt werden. Inhalte können in Kategorien einsortiert werden,
    um die Verwaltung der Inhalte übersichtlicher zu machen.
[*] Passwörter werden jetzt als SHA-512 statt wie vorher als SHA-1 gehasht.
[*] Beim anlegen eines Users ist der Versand der Zugangsdaten optional
[*] Ausgabe von Inhalten auch als PDF, TXT, JSON und als CSV möglich.
[*] Import & Export von einzelnen Datenbanktabellen im JSON-Format möglich. 
Beim Import kann optional auch ein Update bereits existierender Daten erfolgen. Hierbei werden nur die Spalten, die in der importierten Datei enthalten sind, modifiziert
[*] Cache_Lite und PEAR mitgeliefert
